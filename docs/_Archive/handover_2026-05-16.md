# Handover 2026-05-16

## Multi-User Persistence & Storage Isolation 
We've successfully stabilized the backend migration to multi-user storage structures. 

### What was completed:
* **Cookie-Based Auth Context**: Upgraded the client SDK to utilize cookies (`chat-user`) to establish the user identity. It now correctly sets and transmits this token.
* **Database Isolation**: The backend correctly roots database instances into specific profile folders (`data/{chat-user}`), ensuring cross-user isolation for both documents `nDB` and semantic vectors `nVDB`.
* **State Sync Fixes**: `POST /api/chats/:id/messages` now correctly flushes newly created sessions to the disk right away, fixing an issue where new conversations would disappear on page reload.
* **NUI UI Fixes**: Fixed how we save and update Select dropdowns like the TTS Voice setting (relying on `<nui-select>` component API wrapper `.setValue()` rather than naive inner DOM manipulation).
* **Garbage Collection Hooks**: Fixed a vector leakage issue where deleting a conversation via `DELETE /api/chats/:id` dropped the nDB docs and native Image Bucket attachments, but orphaned the semantic embeddings in nVDB. The backend now actively drops these vectors (`embeddingsCol.delete(m.id)`) to maintain database integrity.

### Ongoing Technical Debt:
* **Clunky User Identity Management**: While the storage isolation correctly functions underneath, the user identity token swapping and overall user management remain rigid. This has been added to `docs/features_backlog.md` for a future iteration, as the current system is adequate for "Phase 2" signoff.
